# Спринт 6 — Speaking Rooms + Мобильные + QA

## Цель спринта
Реализовать Speaking Rooms (студент-студент + AI-бот), мобильные приложения (Flutter WebView) и провести финальное QA перед релизом.

---

## 6.1 Speaking Rooms — студент-студент

- [ ] Список доступных комнат с фильтрами (дата, уровень CEFR, тема)
- [ ] Создание комнаты → Google Calendar API → Meet-ссылка автоматически
- [ ] Поля комнаты: название, тема, уровень, язык, дата, длительность (30/45/60 мин), макс. участники
- [ ] Bulk создание — несколько комнат на неделю вперёд за один раз
- [ ] Бронирование места (атомарный счётчик через Redis)
- [ ] Защита от race condition при бронировании
- [ ] Отмена брони в любое время
- [ ] Кнопка «Войти в Google Meet» в назначенное время
- [ ] Комнаты отображаются в Календаре студента (🟢)
- [ ] Напоминания за 1 час и 15 минут (push + email)
- [ ] Оценка сессии после завершения (1–5 звёзд + комментарий)
- [ ] История сессий в личном кабинете
- [ ] Видимость: вся платформа / конкретный марафон

---

## 6.2 Speaking Rooms — AI-бот

- [ ] Запуск AI-сессии мгновенно (без бронирования)
- [ ] Защита от двойного запуска (Redis lock, макс. 1 активная сессия)
- [ ] Выбор темы разговора и уровня сложности
- [ ] OpenAI Whisper STT — транскрипция аудио студента в реальном времени
- [ ] OpenAI GPT — ведение диалога с учётом темы и уровня CEFR
- [ ] OpenAI TTS — синтез аудио ответа AI
- [ ] Показ grammar errors после каждого сообщения студента
- [ ] Ограничение сессии: макс. 30 обменов или 60 минут
- [ ] Итоговый анализ по завершении:
  - [ ] Оценки: fluency, grammar, vocabulary, pronunciation, overall
  - [ ] Сводка ошибок по категориям
  - [ ] Развёрнутая текстовая обратная связь
  - [ ] Рекомендованные темы для следующей практики
- [ ] Pronunciation scoring (OpenAI Whisper / Azure Speech)
- [ ] История AI-сессий в личном кабинете
- [ ] Трекинг токенов и стоимости API (для мониторинга расходов)

---
